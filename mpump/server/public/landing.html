How it moves
A simple loop: make, send, change, resend.

You make a beat, send it as a link, someone else opens it, changes it, and sends it back different. The beat can keep moving without turning into a dead export.

01 / MAKE

Start from a working groove and push it somewhere else.

02 / SEND

Share a playable beat as a link instead of a file or screenshot.

03 / CHANGE

The next person opens the same beat and mutates it in their browser.

04 / RESEND

The groove keeps moving from person to person instead of stopping at export.

Made to be passed on. The next person can open the same groove, change it, and send it back different.
What it is not
Less setup. More motion.

mpump is not trying to be a giant DAW in the browser. It is an instrument. The point is to get into a loop fast, make it yours, and send it on while it still feels alive.

Proof points
Shareable beat links The beat can travel as a URL, not just as an audio bounce.
Live Jam / Live Set Play together without streaming audio. Each browser synthesizes locally.
Offline-capable Open it like a web app and keep working without turning it into a login product.
